## 写前使用规则

1. 正文写作前先看“本章必须持续追踪角色”，确认身份、状态、关系有没有变化。
2. 若本章涉及家庭、街坊、鸣钟司执行层或云阶区，再加载对应“重要配角 / 群体角色”。
3. “低频路人”只作为氛围和旧事证据，不默认进入 `book.config.json` roster，不主动扩充美术槽位。
4. 若后续某个低频路人连续出场或承担剧情功能，再升格到 roster 与角色设定。

## 后续维护规则

- 新角色第一次出现：先判断是否会复用。
- 会复用：加入 `设定/出场角色汇总.md`，必要时加入 `book.config.json > roster`，并补 `设定/角色/{角色名}.md`。
- 只出现一次的街坊 / 背景群众：记录在低频路人区即可，不默认创建美术槽位。
- 群体敌人可以用泛用 roster，例如 `钟卫`，除非出现具名队长或反复对抗者。

## 未来章节新人物处理流程

写正文时如果出现新人物，不要直接一路写下去。按下面四步处理：

1. **先判定功能**：这个人是主线角色、重要配角、群体角色，还是一次性路人。
2. **再判定是否复用**：后续还会出场、会影响选择分支、会生成立绘 / 头像、会改变角色关系的，都视为“会复用”。
3. **会复用就建档**：
   - 加入本文件对应分区；
   - 补 `设定/角色/{角色名}.md`；
   - 若需要前端识别、立绘、头像或 `focusChar`，同步加入 `book.config.json > roster`；
   - 本章写完后重跑 `character_audit.py`，确认没有高 / 中优先级缺失。
4. **不复用只记录**：一次性路人只放进“低频路人 / 暂不入 roster”，不生成美术槽位。

### 快速判断表

| 情况 | 处理 |
|---|---|
| 新女主、新反派、关键盟友 | 立刻加入汇总 + 角色卡 + roster |
| 家人、街坊代表、救命恩人、证人 | 加入汇总；若会再出场或影响视觉表现，再入 roster |
| 鸣钟司具名队长 / 云阶区具名官员 | 加入汇总 + 角色卡；若有反复对抗，入 roster |
| 一群钟卫 / 街坊 / 学舍学生 | 优先用群体角色，不单独建档 |
| 只出现一句的路人 | 低频路人记录即可 |

## 未来章节新任务线处理流程

如果你说的“新任务”是剧情任务线，例如“找账本”“救陆青萝”“潜入鸣钟司”“解锁某个钟权”，处理方式如下：

1. 在本章细纲里写清任务的**目标、阻碍、代价、完成条件**。
2. 写完正文后同步更新：
   - `追踪/伏笔.md`：任务里埋下或推进的线索；
   - `追踪/时间线.md`：任务发生的时间顺序；
   - `追踪/角色状态.md`：任务改变了谁的身份、能力、关系或公众形象；
   - 必要时新增 `追踪/任务线.md`，专门记录长期任务。
3. 如果任务会连续 3 章以上存在，必须在 `追踪/任务线.md` 建条目，避免后面忘记回收。
