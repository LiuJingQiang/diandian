> **服务**：`game-server/` (Express Mock → 演进为真结算服务)
> **对齐参考**：`研究资料/全站存档/api.md` (点点 `api.ddcsai.com` 有状态接口列表)

## 一、后端职责边界

### 🟢 后端**必须权威**（前端不能自主结算）

| 模块 | 职责 | 对应点点 API |
|---|---|---|
| **能量结算** | 扣钟能 · 广告结算 · 首充/回归赠送 | `book/energy_consume` |
| **章节解锁** | 视觉章 17 · 普通章 12 · VIP 折扣 · 灾罪钥匙抵扣 | `device/book_chapter_energy` |
| **选项付费** | 12 钟能扣费 · 限免卡覆盖判定 | `book/energy_consume` |
| **变量结算** | effect apply · sin_layers / bell_authority / dark_mist 等 | 内部 state 存 db |
| **门槛判定** | condition check (AND/OR/嵌套) · 硬门槛回错 | 内部 |
| **钟符购买/贴用** | 30~120 钟能购买 · 1 钟能贴用 · variable_handlers apply | `step1/wardrobe/buy` `step1/wardrobe/wear` |
| **抽卡** | 概率 + 90/10 保底 · 重复卡→碎片 | `card_shop/*` `wish/*` |
| **商店订单** | goods list · create_order · pay verify(签名 sha256/md5) | `step1/pay/*` `pay/check` |
| **VIP 权益** | 8.5 折 · 日发 100 · 侵蚀衰减 -5/日 | `step1/subscribe/*` |
| **任务结算** | 签到 / 阅读 / 广告 (每日重置 · 上限校验) | `new_task_center/*` |
| **结局判定** | Ch19 硬门槛检查 (4/4 证据齐) · Ch20 侵蚀阈值 | 内部 |

### 🔴 后端**不该管**（属于前端）

- UI 渲染 · 动画 · CG 播放 · 音效
- 章节内文本样式 · 视觉主题化命名（"钟能" vs "energy"）
- 首次弹窗 / 引导 tour
- 本地 flag（如"看过 Ch5 引导 tour 了"）

## 二、State 模型（服务端持久化）

### 用户主体 `UserState`

```typescript
interface UserState {
  userId: string;              // uuid or 设备 ID
  bookId: "jinlanchengzui";

  // === 钱包 (currency) ===
  wallet: {
    energy:         number;    // 钟能
    star_dust:      number;    // 香烬砂
    wish_star:      number;    // 案卷星
    love_heart:     number;    // 香火愿力
    option_ticket:  number;    // 免罪符 (剩余分钟数)
    gate_ticket:    number;    // 符券
    card_fragment:  number;    // 案卷残片
  };

  // === 变量池 (对齐 book.config.json 7 vars + 扩展) ===
  variables: {
    sin_layers: number;        // 承罪层数 0-30
    bell_authority: number;    // 回钟权柄 0-200
    case_reading: number;      // 断案眼 0-200
    crowd_stirring: number;    // 煽动力 0-200
    mechanism_wit: number;     // 机变 0-200
    trust_bailing: number;     // 白铃信任 0-300
    trust_luqingluo: number;   // 陆青萝信任 0-300
    trust_tongque: number;     // 铜雀信任 0-300
    alert_hanzhao: number;     // 韩照警觉 0-100 (reversed)
    alley_survival: number;    // 檐灯巷存亡 0-100 (default 60)
    alert_bellsi: number;      // 鸣钟司警觉 0-100
    dark_mist: number;         // 黑雾侵蚀 0-100
  };

  // === 剧情证据 (Set) ===
  evidence: Set<string>;       // ["medicine_ledger", "half_ledger", "father_seal", ...]

  // === 钟符 ===
  talismans: {
    unlocked: Set<string>;     // 已解锁 ID
    equipped: Set<string>;     // 已贴用 ID (最多 3 张同时贴)
  };

  // === 未来案卷 (卡池) ===
  cards: {
    owned: Map<cardId, count>; // 每卡张数
    ssr_pity: number;          // 距 90 保底还差几抽
    sr_pity: number;           // 距 10 保底
  };

  // === 章节进度 ===
  progress: {
    current_chapter: number;   // 1..20
    unlocked_chapters: Set<number>;
    ability_flags: Set<string>;// time_rewind_unlocked / fog_sight_unlocked / citywide_hearing_unlocked
    choice_flags: Set<string>; // path_be1 / path_be2 / evacuated / normal_end / hidden_end_ch20
  };

  // === VIP ===
  vip: {
    active: boolean;
    tier: 7|8|9|10 | null;
    expires_at: timestamp | null;
    daily_bonus_claimed_at: date;
    daily_mist_decayed_at: date;
  };

  // === 任务 ===
  daily: {
    sign_in_streak: number;
    sign_in_claimed_at: date;
    read_time_seconds: number;
    read_milestones_claimed: Set<number>;   // [3,5,10,20,30,60,100,200]
    ad_watched_count: number;               // 上限 15
  };

  // === 商店 ===
  shop: {
    first_pay_used: Set<goodId>;            // 首充档位记录
    new_user_pack_bought: Set<goodId>;      // 新人包 20/21
    weekly_free_claimed_at: date;           // good_id 28
  };

  // === 反流失 ===
  session: {
    last_login_at: timestamp;
    return_gift_pending: boolean;
  };
}
```

## 三、核心 API 契约

### 3.1 认证 & 拉用户状态

```http
POST /api/user/state
Response: UserState (完整)
```

### 3.2 章节 / 段落获取

```http
GET  /api/book/jinlanchengzui/chapter/:ch
Response: {
  chapter: 1,
  title: "三日后的焚街案卷",
  visual_lock: false,
  paragraphs: [...],
  choice_nodes: [...],  // 从 chapters.json 读
  locked: false | { reason: "need_energy", cost: 17 }
}
```

### 3.3 章节解锁（有状态 · 关键）

```http
POST /api/book/chapter/unlock
Body: { chapter: 4 }
Response 200: { unlocked: true, new_energy: 43 }
Response 402: { code: "INSUFFICIENT_ENERGY", need: 17, have: 5 }
```

**服务端逻辑**：
```
1. 校验用户是否已解锁 → 是则返回幂等成功
2. 读章节配置 (visual_lock ? 17 : 12)
3. VIP 折扣: cost = floor(cost * 0.85)
4. 优先扣灾罪钥匙 → 无则扣 energy → 记录 unlocked_chapters
5. 返回新余额
```

### 3.4 选项付费（有状态 · 核心）

```http
POST /api/choice/select
Body: {
  chapter: 4,
  choice_node: "ch4_seal",
  option: "burn_paper"
}
Response 200: { new_state: {...}, effects_applied: {...} }
Response 402: { code: "INSUFFICIENT_ENERGY" }
Response 423: { code: "GATE_FAIL", need: { bell_authority: ">=10" }, have: { bell_authority: 5 } }
```

**服务端逻辑**：
```
1. 从 chapters.json 读该 option 的 { cost, gate, effects, is_ad }
2. gate 判定 (支持 AND / OR / nested):
   - 不通过 → 返回 423
3. is_ad === true:
   - 校验广告 SDK 上报的 token (session/sig)
   - 校验今日广告次数 < 15
4. 否则扣费:
   - 若在 option_ticket 生效期 → 免费
   - 否则 energy -= cost || 报 402
5. apply effects (var 加减、item 获得、flag 设置)
   - clamp 到 [min, max]
   - dark_mist 达 100 → 触发 be_corruption
6. persist state, 返回
```

### 3.5 钟符购买 / 贴用

```http
POST /api/talisman/unlock  { id: "T003" }
POST /api/talisman/wear    { id: "T003", equip: true }
```

**服务端逻辑**：
- `unlock`: 校验 acquire 条件（shop cost/story chapter/gacha）→ 扣费 → 加入 unlocked Set
- `wear`: 校验 wear_cost（钟能 or 符券）+ conditions（"favor_x >= N" 等）→ apply handlers → 加入 equipped
- 卸下：反 apply handlers

### 3.6 未来案卷抽卡

```http
POST /api/gacha/pull  { pool: "future_case", count: 1|10 }
Response: {
  results: [{ id: "F004_SR", rarity: "SR", is_new: true }, ...],
  new_pity: { ssr: 45, sr: 3 }
}
```

**逻辑**：
- 扣 wish_star（10 / 100）
- 逐抽 roll：
  - `sr_pity >= 10` 则强制 SR+
  - `ssr_pity >= 90` 则强制 SSR
  - 否则按 SSR 2% / SR 12% / R 40% / N 46%
- 重复卡 → 转 card_fragment
- 更新 pity

### 3.7 商店 / 支付

```http
GET  /api/shop/goods                  → 返回全档位（读 shop_goods.json）
POST /api/shop/order/create            { good_id: 103 } → 创建订单 + 微信/支付宝 sign
POST /api/shop/order/verify            { order_id, sign } → 回调校验 → 发货
```

**发货逻辑**：
- 能量包：`energy += base + (first_pay_used?0:first_pay_bonus)`
- VIP：写入 vip.tier / vip.expires_at + 送 1000 钟能礼包
- 陆青萝药包：立即 apply detox effects
- 香火愿力包：加 love_heart + alley_survival

### 3.8 任务系统

```http
GET  /api/task/daily                → 返回签到/阅读/广告状态
POST /api/task/sign_in              → 领签到
POST /api/task/read_milestone       { minutes: 30 } → 领里程碑
POST /api/task/ad_reward            { ad_token }    → 校验并领 12 钟能
POST /api/task/return_gift          → 3 天未登录回归礼包
```

## 四、变量结算引擎（重点）

### 4.1 effect apply

```typescript
function applyEffect(state, effect) {
  for (const [key, delta] of Object.entries(effect)) {
    if (key === 'item')     state.evidence.add(delta);
    else if (key === 'item_all') delta.forEach(i => state.evidence.add(i));
    else if (key === 'flag') state.progress.choice_flags.add(delta);
    else {
      const range = getVarRange(key);              // 从 variables.json 读
      const target = state.variables[key] ?? 0;
      state.variables[key] = clamp(target + delta, range.min, range.max);
    }
  }
  // 触发衍生: dark_mist >= 100 → BE, alley_survival <= 0 → BE
  checkGlobalFailures(state);
}
```

### 4.2 condition check（支持嵌套 AND/OR）

```typescript
function checkCondition(state, cond) {
  if (cond.op === 'AND') return cond.clauses.every(c => checkCondition(state, c));
  if (cond.op === 'OR')  return cond.clauses.some(c  => checkCondition(state, c));
  if (cond.item_has)     return state.evidence.has(cond.item_has);
  if (cond.item_has_any_of) return cond.item_has_any_of.some(i => state.evidence.has(i));
  if (cond.flag_has)     return state.progress.choice_flags.has(cond.flag_has);
  if (cond.evidence_count) return compare(state.evidence.size, cond.evidence_count);
  // 变量比较
  for (const [key, expr] of Object.entries(cond)) {
    if (typeof expr === 'object' && 'op' in expr) {
      return compare(state.variables[key], expr);
    }
  }
}
```

## 五、反作弊 & 数据完整性

1. **签名**：POST body AES-CBC + `sign = md5(...APP_SECRET)`（对齐点点）
2. **幂等**：所有扣费接口带 `client_op_id`，服务端去重
3. **状态版本号**：`UserState.version` 乐观锁，前端提交 stale version 拒绝
4. **广告 token 校验**：广告 SDK server-to-server callback
5. **单日操作上限**：广告 15/日、签到 1/日、每章解锁一次
6. **变量边界**：所有 apply 后 clamp 到 [min, max]
7. **回滚**：订单未回调支付成功前不发货；发货失败入补偿队列

## 六、错误码

| Code | HTTP | 含义 |
|---|---|---|
| `INSUFFICIENT_ENERGY` | 402 | 钟能不足 |
| `INSUFFICIENT_CURRENCY` | 402 | 其他货币不足 |
| `GATE_FAIL` | 423 | 门槛未通过（回 need/have 供前端展示提示）|
| `CHAPTER_LOCKED` | 423 | 前置章节未通关 |
| `AD_QUOTA_EXCEEDED` | 429 | 广告日上限 |
| `TASK_ALREADY_CLAIMED` | 409 | 任务已领 |
| `STATE_VERSION_STALE` | 409 | 状态版本冲突 |
| `INVALID_SIGN` | 401 | 签名错误 |
| `ITEM_NOT_OWNED` | 404 | 尝试贴未拥有的钟符 |

## 八、配置热加载 & 版本管理
